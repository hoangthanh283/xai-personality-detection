# Detailed Classification Reports — All Averaging Variants

Generated by `scripts/detailed_classification_report.py` via sklearn.

### How to read these tables

We report **all** standard averaging variants for transparency, but you should 
**use Macro F1 to compare models and make decisions**. Rationale:

- **Macro F1** (`f1_macro`) — **PRIMARY**. Arithmetic mean of per-class F1. 
  Each class counts equally. Honest on imbalanced data.
- **Accuracy** / **Micro F1** (`f1_micro`) — Mathematically equal to accuracy for 
  single-label classification. Inflated by majority class on imbalanced tasks.
- **Weighted F1** (`f1_weighted`) — F1 averaged weighted by class support. 
  Dominated by majority class → same bias as accuracy. Do not use for ranking.
- **Balanced Accuracy** (`bal_acc`) — Mean recall per class. Intuitive twin of 
  Macro F1 when you care about catching every class equally.
- **Per-class P / R / F1 / support** — Drill-down for failure analysis (e.g., 
  detecting majority-class collapse where a class has recall ≈ 0).

> **Decision rule**: when two models' accuracies differ by < 5 pp but macro F1 
> differs by > 3 pp, trust macro F1.
